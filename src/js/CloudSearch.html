CloudSearch

It encapsulates a set of methods and attributes required to perform search on your data in CloudApp to retrieve desired set of relevant results.

 

Constructor

CB.CloudSearch( tableName, [CB.SearchQuery], [CB.SearchFilter])

Argument
tableNames
Array/string The name of the table(s) you want to search on.
CB.SearchQuery
CB.SearchQuery (Optional)CB.SearchQuery object for search.
CB.SearchFilter
CB.searchFilter (Optional)CB.SearchFilter object for search.
var cs = new CB.CloudSearch("Student"); //create a new filter cs.searchFilter = new CB.SearchFilter(); //create a new query cs.searchQuery = new CB.SearchQuery();
 

SearchQuery.searchOn( columns, queryText, [fuzziness], [all_words], [match_percent], [priority] )

Used to perform a search query.

Arguments
columnName
string / Array The column on which the contraint is to be applied
queryText
stringA text query
fuzziness
[Optional] integerCan be set to 0, 1, 2. Default is 0. This parameter sets the fuzziness on search.
all_words
[Optional] stringCan take only value - true. If set the search will contain all the words in query. For example, If your search query is "Brown Fox". Results will contain both the words "Brown" AND "Fox".
match_percent
[Optional] stringCan take only value - %. For ex : "75%". If set the search will return all the results which matches 75%.
priority
[Optional] integerIf set this query will have a higer priority over all other queries.
Returns

Same SearchQuery object, which can be helpful in chaining search functions.

var cs = new CB.CloudSearch('Student'); cs.searchQuery = new CB.SearchQuery(); cs.searchQuery.searchOn('resume', 'Java'); cs.search({ success : function(list){ //list is an array of CloudObjects },error : function(error){ //error } });

SearchQuery.bestColumns( columns, queryText, [fuzziness], [all_words], [match_percent], [priority] )

This function is most useful when you are searching for multiple words best found in the same field. For instance “brown fox” in a single field is more meaningful than “brown” in one field and “fox” in the other. For example : If you have two records, ID Sample 1 Sample 2 1 Brown Fox Up the hill 2 Brown Fox up the hill
In this case, If you search for brown fox then Record with ID 1 will be more relevant than record with ID 2

Arguments
columnNames
Array The column on which the contraint is to be applied
queryText
stringA text query
fuzziness
[Optional] integerCan be set to 0, 1, 2. Default is 0. This parameter sets the fuzziness on search.
all_words
[Optional] stringCan take only value - true. If set the search will contain all the words in query. For example, If your search query is "Brown Fox". Results will contain both the words "Brown" AND "Fox".
match_percent
[Optional] stringCan take only value - %. For ex : "75%". If set the search will return all the results which matches 75%.
priority
[Optional] integerIf set this query will have a higer priority over all other queries.
Returns

Same SearchQuery object, which can be helpful in chaining search functions.

var cs = new CB.CloudSearch('Student'); cs.searchQuery = new CB.SearchQuery(); cs.searchQuery.bestColumns(['Sample1','Sample2'], 'Brown Fox'); cs.search({ success : function(list){ //list is an array of CloudObjects },error : function(error){ //error } });

SearchFilter.or( searchFilter )

It adds a logical OR contriant to SearchFilter object for finding the set of the data satisfying conditions of SearchFilter

Arguments
SearchFilter
SearchFilter Object
Returns

It returns a SearchFilter object which contains OR'ed query of SearchFilter object.

//find all Student such that 10<age<18 //create first searchFilter. var searchFilter1 = new CB.SearchFilter(); searchFilter1.lessThan('age', 18); //create second search Filter. var searchFilter2 = new CB.SearchFilter(); searchFilter2.greaterThan('age', 18); //or both of these. var searchFilter = new CB.SearchFilter(); searchFilter.or(searchFilter1); searchFilter.or(searchFilter2); //attach OR'ed searchFilter to CloudSearch. var cs = new CB.CloudSearch('Student'); cs.searchFilter = searchFilter; //search term cs.searchQuery = new CB.SearchQuery(); cs.searchQuery.searchOn('name','Dan'); cs.search({ success: function(list){ //list is an array of CloudObjects }, error: function(err) { //Error in retrieving the data. } });

CloudSearch.search( [callback] )

Searches the database on the constraints provided by the CloudSearch object and gets the result as an array of CloudObject objects.

If no. of documents to be returned is not specified using limit() then at max 20 documents are returned in each query by default.

Arguments
callback
Object [optional] if provided must have success and error functions to handle respective response.
Returns

It returns an array of CloudObject objects or an error object if the opertion is successful or fails respectively along with JQuery promise (if callback is not provided).

cs.search({ success: function(list){ //list is an array of CloudObjects }, error: function(err) { //Error in retrieving the data. } });
